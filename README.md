# Математическая модель и обработка результатов голосования
Требуется сравнить результаты
- Метод относительного большинства
- Метод абсолютного большинства
- Метод минимальной суммы мест
- Метод эталонов

## Математическая модель голосования
Результаты голосования представлены списком имён кандидатов и целочисленной квадратной матрицей, где в позиции [r,c] записано число голосов, отданных за место r кандидату c.
Для генерации челочисленной квадратной матрицы голосов используется клас RandomElection. Параметром генерации матрицы является количество выборшиков и список кандидатов. Для каждого выборшика заполняет бюллетень, каждый бюллетень представляет собой случайную перестановку всех кандидатов. По результатам обработки всех бюллетеней составляется квадратная матрица голосов.

# Метод относительного большинства
Побеждает кандидат, набравший наибольшее число голосов. 

# Метод абсолютного большинства
В начале проходит простой тур голосования. Все выборщики подают свои голоса за одного наиболее предпочитаемого для них кандидата. Если какой-то кандидат набирает больше всех голосов, а точнее больше половины от общего числа выборщиков, то он и становится абсолютным победителем. Иначе проводится 2 тур голосования, в который проходят два кандидата, набравших наибольшее число голосов в первом туре. Во втором туре, уже по правилу относительного большинства или по обычному правилу, для двух кандидатов, проводится голосование. После 2 тура окончательно определяется победитель.

# Метод минимальной суммы мест
- Каждый выборщик дает i очков кандидату, поставленному на i-ое место
- Все набранные очки суммируются
- Побеждает кандидат, набравший минимальную сумму

# Метод эталонов
Содержательно, метод эталонов ставится следующим образом: требуется найти минимальное расстояние от наилучшего эталона или максимальное расстояние от наихудшего эталона.
В качестве наилучшего эталона также принимается фиктивный кандидат, за которого проголосовали все избиратели.
Очевидно, что наилучшему эталону соответствует вектор {w, 0, 0,…., 0}
